# IDNA Authority Boundary

## What IDNA Is

IDNA defines the authoritative semantic domain within which system states,
actions, and invariants are evaluated for admissibility.

It establishes the boundary at which identity, provenance, and meaning are
normalized so that admissibility decisions are coherent and unambiguous.

## Role in the Architecture

IDNA does not execute actions or evaluate invariants itself.
It provides the semantic context against which admissibility is evaluated by
architectural authorities such as DREA.

## Out of Scope / Not Claimed

IDNA does not define identity mechanisms, credential formats, data models,
or implementation details for normalization or provenance handling.
