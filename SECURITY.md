# Security Policy

## Reporting Security Vulnerabilities

If you discover a security vulnerability in this project, please report it privately before disclosing it publicly.

### Reporting Process

- Email: security@novafuse.org
- Include detailed description of the vulnerability
- Provide steps to reproduce if possible
- Allow us reasonable time to address the issue before public disclosure

### Response Time

We will acknowledge receipt of your report within 48 hours and provide a detailed response within 7 days.

## Supported Versions

Only the latest version of this reference architecture receives security updates.

## Security Best Practices

This architecture incorporates security best practices including:

- Formal verification of safety invariants
- Threat modeling and risk assessment
- Evidence-based compliance verification
- Secure interface definitions

For detailed security considerations, see `governance/threat-model.md`.
